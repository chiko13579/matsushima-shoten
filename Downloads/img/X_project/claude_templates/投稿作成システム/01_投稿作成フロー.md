# 📝 X投稿作成フロー

このファイルは投稿作成の全体的な流れを示しています。必ずこの順番で進めてください。

---

## 🎯 投稿作成の基本フロー

### STEP 1: 投稿の種類を決定
`03_投稿パターン詳細.md`を必ず読み込んで、以下の手順で進める
1. **投稿種類の選択と目的の把握**
   選択した投稿種類の目的を必ず把握してから作成に進む：
   
   - **「相手が知りたいことに回答する」**
     目的：この方法だとうまくいきそう!試してみたい!それ知らなかった！って思ってもらう
   
   - **「理想の未来と解決策パターン」**
     目的：自分もこうなりたい!! こうなれるかもとワクワクする
   
   - **「ターゲットの不安を払拭する。肯定する/安心させる」**
     目的：自分もこうなりたい!! こうなれるかもとワクワクする
   
   - **「ストーリー」やその他の種類**
     目的：`03_投稿パターン詳細.md`で確認

2. **質問・テーマの選択**
   - 選んだ投稿種類に応じて：
     - 「相手が知りたいことに回答する」→ `04_質問データベース.md`から質問を選択
     - 「理想の未来と解決策」→ `05_理想の未来データベース.md`と`06_解決策データベース.md`から選択
     - その他の種類もそれぞれ対応するファイルから選択

3. **パターン・構造の判定**
   - 選んだ投稿種類に応じて使用する構造を確認：
     - **「相手が知りたいことに回答する」**→ 投稿スタイル6パターン（パターン1〜6）から最適なものを選択
     - **「理想の未来と解決策パターン」**→ 専用の構造を使用（理想の状態→解決策→効果・結果→励まし）
     - **「ターゲットの不安を払拭する」**→ 専用の構造を使用
     - **その他の投稿種類**→ `03_投稿パターン詳細.md`で定義された対応する構造を使用
   - 選択した構造・パターンの必須要素を確認

### STEP 2: 文字数を決定
投稿の種類に応じて適切な文字数を設定：

- **「ターゲットの不安を払拭する。肯定する/安心させる」** → 120文字から200文字程度
- **「相手が知りたいことに回答する」** → 300文字以上500文字以内
- **「理想の未来と解決策パターン」** → 250文字から400文字

### STEP 3: 質問の心理を分析する
選んだ質問について、**その質問の裏にある本当の気持ち・知りたいことを深く分析**

#### 📚 分析フレームワーク：
**表面的**：何を聞いているか
**本当の気持ち**：なぜその質問をするのか
- 不安・恐怖・疑い・焦り・劣等感など具体的に
**深層心理**：`07_不安・共感フレーズデータベース.md`から該当する心の声を参照
**回答に含めるべき要素**：
- 不安の解消
- 安心感の提供
- 具体的な希望・光の提示
- 「自分にもできる」という確信

#### 分析例：
**「値上げのタイミングってどう判断する？」**
- **表面的**：判断基準を知りたい
- **本当の気持ち**：
  - 値上げしたら切られるんじゃないか不安
  - 自分の価値に確信が持てない
  - 今の単価のままじゃ生活できないけど怖い
  - 成功例を知って安心したい
- **深層心理**：「値上げ交渉したら切られるかも」「友達に時給聞かれても恥ずかしくて答えられない」
- **回答に含めるべき要素**：
  - 値上げへの恐怖心への共感
  - 切られない値上げ方法
  - 成功の具体例と安心感
  - 段階的なアプローチで希望提示

⚠️ **重要：この分析を基に、表面的な回答ではなく深層心理に響く投稿を作成する**

### STEP 4: 投稿を作成（内容重視）
STEP1で選択した構造・パターンの必須要素を`03_投稿パターン詳細.md`で確認しながら作成

**⚠️ 重要：投稿目的を常に意識**
- 選択した投稿種類の目的を達成するための内容になっているか常にチェック
- 例：「相手が知りたいことに回答する」なら「試してみたい！」と思わせる具体性があるか
- 例：「理想の未来と解決策」なら「こうなりたい！」とワクワクする希望があるか

**⚠️ 重要：投稿作成マスターガイドの活用**
`02_投稿作成マスターガイド.md`を参照して以下を必ず含める：
- **甘い情報・希望を感じる要素**：例文集から適切なものを選択
- **デザイン業界特有の実践的な情報**：例文集から適切なものを選択

**⚠️ 重要：実体験ベースの制約**
- ちこが実際にやったことのみ回答に含める
- 初心者には難しすぎる方法は避ける
- 「売上UP実績」「効果測定」など高度な手法は使わない
- **実際の営業先**：制作会社・マーケティング会社・ランサーズ・クラウドワークス
- 基本的な営業・単価交渉を中心とした現実的なアドバイス

**⚠️ 重要：背景情報の反映**
- `04_質問データベース.md`の【ターゲットの現実的な背景情報】を必ず参照
- クラウドソーシングで消耗している現実を踏まえる
- 「営業しても全然返信来ない」などのリアルな悩みに共感
- みんながやっている方法 vs 実は効果的な方法の対比構造を活用

**⚠️ 重要：この段階では冒頭は仮でOK、内容の質を最優先に！**
**⚠️ 重要：STEP 3の心理分析を必ず反映させる！**

⚠️ **重要：質問回答系投稿の具体性レベル**
質問に回答する投稿では、必ず以下レベルの具体性を保つこと：

**❌ 抽象的すぎる例：**
- 「営業を頑張りました」
- 「単価を上げました」
- 「継続案件を取りました」

**✅ 具体的で実践的な例：**
- 「営業先を『広告代理店』『制作会社』に絞った」
- 「提案文に『前回の案件で問い合わせ3倍になりました』と売上UP実績を必ず入れた」
- 「初回は相場の8割で受けて、2回目で1.5倍に上げる（具体例：5万→7.5万→10万〜）」

**必須要素：**
- 具体的な業界名・企業タイプ
- 実際の金額・数字
- 具体的な文言・フレーズ例
- 明日から行動できるレベルの詳細度
- タイミングや手順の明確化

### STEP 5: 冒頭強化
投稿の内容が完成した後、`02_投稿作成マスターガイド.md`の冒頭強化ガイドを参照：

1. **冒頭スタイルを選択** - 質問内容に応じて体験談風 vs 通常を判定
2. **選択したスタイルに応じて冒頭を作成**：
   - **体験談風**：パターン例を参考に作成
   - **通常**：心理フック・強い言葉・深い共感フレーズ例から組み合わせ
3. **自然さを最優先** - 詰め込みすぎず、無理に入れない

### STEP 6: 冒頭のみ修正
内容はそのままで、冒頭1-2文のみを強化された内容に差し替える

### STEP 7: 品質チェック
投稿作成後、以下の観点で品質確認：

#### 1. 冒頭の強さ調整
- `02_投稿作成マスターガイド.md`の冒頭強化ガイドを参考にしつつ、ウザくならない程度になっているか
- 心理フック・強い言葉・深い共感のバランスは適切か

#### 2. ちこの実体験との整合性
- `08_ちこ体験談データベース.md`と矛盾していないか
- 嘘のエピソードを作っていないか

#### 3. ちこらしい文体
- `09_ちこ口調データベース.md`を参照して確認
- 「なぁって」「めっちゃ」などの自然な関西弁が使われているか
- 優しく共感力のある語り口になっているか

### STEP 8: 最終チェックリスト確認
**`02_投稿作成マスターガイド.md`の最終チェックリストで、記載されている全ての項目を確認してください。**

**確認の流れ：**
1. 基本要件をチェック
2. 自然な文章の書き方をチェック  
3. ちこらしさをチェック
4. 全項目がOKになったら投稿完成！

## 🔄 投稿作成の実例

**例：「月30万達成するために、何したの？」への回答**

1. **種類決定** → 相手が知りたいことに回答する
   **目的把握** → 「この方法だとうまくいきそう!試してみたい!それ知らなかった！」と思ってもらう
2. **質問選択** → 「月30万達成するために、何したの？」
3. **構造選択** → 投稿スタイル6パターンから「パターン4（悩み解決・具体的手法）」を選択
4. **文字数** → 300文字
5. **心理分析** → 表面的な方法論だけでなく、不安や焦りの解消も含める
6. **作成** → パターン4の必須要素を含め、「試してみたい！」と思わせる具体性を重視
7. **確認** → チェックリスト全項目OK

このフローに従えば、一貫性のある質の高い投稿が作成できます。
