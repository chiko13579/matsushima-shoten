# 📖 X自動投稿システム 完全運用マニュアル

## 🎯 システム概要

### システム構成
```
投稿作成 → Googleスプレッドシート → n8n → X投稿 → 結果更新
```

### 処理フロー
1. **投稿準備**: スプレッドシートに投稿内容を追加
2. **承認**: ステータスを「承認済み」に変更  
3. **自動実行**: n8nが15分間隔でチェック・投稿
4. **結果更新**: 投稿後、自動でステータス・URLを更新

## 📝 日常運用手順

### 1. 新しい投稿の追加

#### 1-1. スプレッドシートに投稿を追加
```
手順:
1. Googleスプレッドシート「X投稿管理システム」を開く
2. 新しい行に以下を入力:
   - ID: 連番(自動採番可)
   - 投稿内容: 作成した投稿テキスト
   - 投稿種類: [不安払拭/質問回答/理想の未来/その他]
   - ステータス: [下書き] ← 最初は必ず「下書き」
   - 投稿予定日時: YYYY/MM/DD HH:MM 形式
   - 他の列は空白でOK
```

#### 1-2. 投稿内容の確認
```
チェック項目:
✅ 文字数: 280文字以内（日本語は全角1文字=2カウント）
✅ ハッシュタグ: 適切に設定済み
✅ 誤字脱字: 確認済み
✅ 不適切な内容: なし
✅ リンク: 正常に機能する
```

#### 1-3. 承認・スケジュール設定
```
承認手順:
1. 内容を最終確認
2. ステータスを「下書き」→「承認済み」に変更
3. 投稿予定日時が正しいことを確認
4. 保存（Ctrl+S）
```

### 2. 投稿状況の確認

#### 2-1. 投稿実行の確認
```
確認項目:
- 投稿予定時刻から30分以内に実行されているか
- ステータスが「投稿完了」に更新されているか
- 投稿URLが記録されているか
- 実際投稿日時が記録されているか
```

#### 2-2. エラー対応
```
エラー時の確認:
1. ステータスが「エラー」になっていないか確認
2. メモ欄にエラーメッセージがないか確認
3. エラーがある場合は後述のトラブルシューティングを参照
```

## ⚠️ トラブルシューティング

### 1. よくある問題と対処法

#### 問題1: 投稿されない
```
原因と対処:
❌ ステータスが「承認済み」になっていない
   → ステータスを確認・修正

❌ 投稿予定日時が未来になっている  
   → 日時を現在時刻以前に設定

❌ 投稿内容が空
   → 投稿内容を確認・入力

❌ ワークフローが無効
   → n8nでワークフローを「Active」に設定
```

#### 問題2: 投稿でエラーになる
```
エラーメッセージ別対処:

"Authentication failed"
→ X APIの認証情報を確認・再設定

"Rate limit exceeded"  
→ API制限に到達。時間をおいてリトライ

"Duplicate content"
→ 同じ内容の投稿が既に存在。内容を変更

"Content too long"
→ 280文字以内に短縮
```

#### 問題3: スプレッドシートが更新されない
```
対処法:
1. Google Sheets APIの認証を確認
2. スプレッドシートの共有権限を確認  
3. n8nのGoogle Sheets認証情報を再設定
```

### 2. 緊急時対応

#### システム停止が必要な場合
```
停止手順:
1. n8nにログイン
2. ワークフローの「Active」をOFFに設定
3. 問題解決後、再度「Active」をONに設定
```

#### 投稿の緊急削除
```
削除手順:
1. Xで該当投稿を手動削除
2. スプレッドシートのステータスを「削除済み」に変更
3. メモ欄に削除理由を記載
```
