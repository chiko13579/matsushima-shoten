# Google Sheets API 設定手順

このツールでGoogleスプレッドシートに結果を保存するための設定手順です。

## 1. Google Cloud Platform プロジェクトの作成

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセス
2. 新しいプロジェクトを作成、または既存のプロジェクトを選択
3. プロジェクト名を入力（例：`web-company-research`）

## 2. Google Sheets API を有効化

1. Google Cloud Console で「APIとサービス」→「ライブラリ」を選択
2. 「Google Sheets API」を検索
3. 「Google Sheets API」をクリック
4. 「有効にする」ボタンをクリック

## 3. サービスアカウントの作成

1. 「APIとサービス」→「認証情報」を選択
2. 「認証情報を作成」→「サービスアカウント」を選択
3. サービスアカウント情報を入力：
   - **サービスアカウント名**: `web-company-scraper`
   - **サービスアカウントID**: 自動生成される
   - **説明**: `Web制作会社調査ツール用`
4. 「作成して続行」をクリック
5. ロールの設定：
   - 「ロールを選択」で「編集者」を選択
6. 「完了」をクリック

## 4. 認証キーの生成とダウンロード

1. 作成したサービスアカウントをクリック
2. 「キー」タブを選択
3. 「鍵を追加」→「新しい鍵を作成」を選択
4. キーのタイプは「JSON」を選択
5. 「作成」をクリック
6. JSONファイルが自動でダウンロードされます

## 5. 認証ファイルの配置

1. ダウンロードしたJSONファイルの名前を `credentials.json` に変更
2. このファイルをプロジェクトのルートディレクトリに配置
   ```
   my_list_project/
   ├── credentials.json  ← ここに配置
   ├── index.js
   ├── package.json
   └── ...
   ```

## 6. スプレッドシートの共有設定

スプレッドシートにアクセスするため、以下の手順が必要です：

### 新規スプレッドシートの場合
ツールが自動で作成するため、特別な設定は不要です。

### 既存スプレッドシートの場合
1. Googleスプレッドシートを開く
2. 右上の「共有」ボタンをクリック
3. サービスアカウントのメールアドレスを追加
   - メールアドレスは `credentials.json` の `client_email` フィールドで確認できます
   - 例：`web-company-scraper@your-project.iam.gserviceaccount.com`
4. 権限を「編集者」に設定
5. 「送信」をクリック

## 7. 使用方法

設定完了後、ツールを実行すると：

```bash
npm start
```

保存オプションで「2. Googleスプレッドシート」または「3. 両方」を選択できます。

### 新規スプレッドシート作成
- 既存スプレッドシートIDを空白のままEnterキーを押す
- 新しいスプレッドシートが作成され、URLが表示されます

### 既存スプレッドシートに追加
- スプレッドシートのURLから ID を取得
- 例：`https://docs.google.com/spreadsheets/d/SPREADSHEET_ID/edit`
- `SPREADSHEET_ID` 部分をコピーして入力

## トラブルシューティング

### よくあるエラー

#### `credentials.json が見つかりません`
- ファイルがプロジェクトルートに配置されているか確認
- ファイル名が正確に `credentials.json` になっているか確認

#### `403 Forbidden`
- Google Sheets API が有効化されているか確認
- サービスアカウントに適切な権限があるか確認
- 既存スプレッドシートの場合、共有設定を確認

#### `404 Not Found`
- スプレッドシートIDが正しいか確認
- スプレッドシートがサービスアカウントと共有されているか確認

### サポート

問題が解決しない場合は、以下を確認してください：
1. Google Cloud Console でプロジェクトが正しく設定されているか
2. APIの使用量制限に達していないか
3. インターネット接続が安定しているか

## セキュリティ注意事項

⚠️ **重要**: `credentials.json` ファイルは機密情報です
- Gitリポジトリにコミットしないでください
- 他人と共有しないでください
- 使用後は安全に管理してください
