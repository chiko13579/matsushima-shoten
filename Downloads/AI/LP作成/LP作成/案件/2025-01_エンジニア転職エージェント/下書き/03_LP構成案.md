# LP構成案 - エンジニア向け転職エージェント

---

## 📋 **基本方針**

### **LF8設計**
```
メインLF8-3(恐怖回避: キャリアの停滞・市場価値の低下)
+ サブLF8-8(社会的承認: エンジニアとして成長)
+ サブLF8-6(優越感: 周りに負けたくない)
```

### **トーン**
```
共感、誠実、対等な立場
※押し売り感を出さない
※「今すぐ転職しなくてもOK」を繰り返し伝える
```

### **ポジショニング**
```
「キャリアの壁打ち相手」
エンジニア専門だから、技術の話が通じる
押し売りしない、情報提供重視のエージェント
```

### **ゴール**
```
登録してもらい、つながりを持つこと
(今すぐ転職させることではない)
```

### **1. ファーストビュー(FV)**

#### **目的**
- ターゲットに「自分のことだ」と思わせる
- LF8-3(恐怖回避)を匂わせる
- 「今すぐ転職じゃない」ことを最初から伝える

#### **構成要素**

**【キャッチコピー】**
```
「このままでいいのか...」
そのキャリアの不安、技術に詳しいプロに相談してみませんか?
```

**【サブコピー】**
```
転職するかどうかは、まず市場価値を知ってから決めよう

エンジニア専門だから、技術の話が通じる
押し売りしません。まずは、キャリアの壁打ち相手として
```

**【画像】**
- パソコンに向かって考え込むエンジニア
- または、カフェでノートPCを見ながら考えている様子

**【CTA】**
```
[登録3分、相談無料]
まずは市場価値を知る →
```

**【信頼要素】**
```
✓ エンジニア経験者がアドバイザー
✓ 登録者○○名突破
✓ 平均年収アップ額○○万円
```

### **2. 問題提起(共感パート)**

#### **目的**
- ターゲットの現状に共感する
- 「自分のことだ」と思わせる
- 漠然とした不安を言語化する

#### **構成**

**【見出し】**
```
仕事に関して、こんな風に感じたことはありませんか？
```

**【チェックリスト形式】**
```
□ 今の会社、悪くはないけど...このままでいいのか?
□ 自分の市場価値が分からず、不安
□ 同期が転職して年収が上がってるの、正直羨ましい
□ 最新技術に触れられず、スキルが伸び悩んでる
□ でも、転職する明確な理由があるわけじゃない
□ 「今動くべきか、待つべきか」判断できない
□ 転職エージェントって、押し売りされそうで怖い
```

**【共感の一言】**
```
今の会社が嫌なわけじゃない。
でも、「このまま」でいいのか、漠然とした不安がある。

その不安、すごく分かります。

でも、その不安を放置すると、
取り返しがつかないことになります。
```

### **3. 成功事例(ポジティブ体験)**

#### **目的**
- 「こんなに良くなるんだ!」と希望を見せる
- 「まだ転職するか決めてない段階」から相談できることを示す
- 北達人方式: ポジティブ体験 → 好奇心 → 問題の教育

#### **構成**

**【見出し】**
```
実際に利用された方の声
```

**【体験談1: 29歳エンジニア・市場価値を知ってから転職】**
```
「市場価値を知ったら、自信を持って転職できました」

漠然と「転職した方がいいのかな...」と思ってたけど、
自分のスキルが他社でどう評価されるか分からなくて。

でも、[サービス名]で相談したら、
「あなたのスキルなら、年収+150万円は狙えます」と
具体的に教えてもらえて。

市場価値を知ったことで、自信を持って転職活動できました。

結果、年収が180万円アップ。
「あの時相談してて良かった」と心から思います。

(29歳・Web系エンジニア・年収450万→630万円)
```

**【体験談2: 32歳エンジニア・相談だけで転職しなかった人】**
```
「転職しませんでしたが、相談して良かったです」

「このままでいいのか不安」で登録しました。

アドバイザーに市場価値を聞いたら、
「今の会社の待遇、実は結構良いですよ。
 転職するなら、スキルをもう少し磨いてからの方がいいかも」
と正直に言ってもらえて。

押し売りされると思ってたけど、
むしろ「今は動かない方がいい」とアドバイスしてくれた。

結果、今の会社に残ることにしましたが、
「情報を知った上で決めた」ので、不安がなくなりました。

いい求人が出たら連絡もらえるので、
つながりを持っておいて良かったです。

(32歳・インフラエンジニア)
```

**【疑問を作る(好奇心喚起)】**
```
「でも、なんで市場価値を知るだけで、不安が減るの?」

実は、キャリアの不安の正体は
「情報がない」ことなんです。
```

### **4. 問題の教育(キャリアの不安を放置する危険性)**

#### **目的**
- 「このまま」の危険性を認識させる
- LF8-3(恐怖回避)を刺激する
- でも、恐怖訴求しすぎない(バランス重要)

#### **構成**

**【見出し】**
```
その「このままでいいのか」の不安、
放置すると取り返しがつかないことに
```

**【問題1: キャリアには"動ける時期"がある】**
```
「もう少し様子を見てから...」
「後でいいや」

その「後で」が、実は一番危ない。

なぜなら、
キャリアには"動ける時期"があるから。

【20代後半】
ポテンシャル採用がまだある
選択肢が多い

【30代前半】
経験+成長性で判断される
まだ選べる

【30代後半】
即戦力+マネジメント経験が必須
選択肢が急激に狭まる

「あと数年待ってから...」
その数年が、キャリアの分岐点です。
```

**【問題2: 市場価値は、30代で決まる】**
```
エンジニアの市場価値は、
20代でどんな経験を積むかで決まります。

【良い例】
20代: モダンな技術スタック、成長企業
30代: 高い市場価値、選択肢が多い

【悪い例】
20代: レガシーな環境、技術的負債
30代: 市場価値が低い、選択肢が少ない

今、あなたはどちらに向かってますか?
```

**【問題3: 技術トレンドに乗り遅れる】**
```
技術の変化は、想像以上に速い。

5年前のスタックが、もう古い。
「これができます」と思っていたスキルが、
企業からは「ちょっと古いね」と言われる。

レガシーな環境にいると、
気づいたら市場価値が下がっている。

「自分は大丈夫」
そう思っている人ほど、危ない。
```

**【問題4: 情報がないまま判断している】**
```
「今の会社、まあまあかな」

でも、本当に?

他の会社の給料は?
開発環境は?
技術スタックは?
働き方は?

比較対象がない状態で、
「今の会社で良い」と思い込んでいませんか?

情報がないまま決めると、
「あの時動いておけば...」と後悔します。
```

**【恐怖訴求の締め】**
```
「このままでいいのか...」

その不安の正体は、
「情報がない」からです。

情報がないまま1年、2年...
気づいたら、動けなくなっている。

だから、まずは情報を持つことから始めましょう。
```

### **5. 目的の教育(まずは市場価値を知ることから)**

#### **目的**
- 「今すぐ転職じゃない」ことを明確に伝える
- 「情報を持つ」ことの重要性を伝える
- ハードルを下げる

#### **構成**

**【見出し】**
```
転職するかどうかは、
まず市場価値を知ってから決めればいい
```

**【ポイント1: 情報がない状態での決断は危険】**
```
今の会社に「残る」「転職する」を決めるのに、
一番大事なのは「情報」です。

✅ 自分の市場価値は?
✅ 他にどんな選択肢がある?
✅ 今動くべき? 待つべき?

これが分かって初めて、正しい決断ができます。

転職するかどうかは、その後決めればいい。
```

**【ポイント2: 市場価値を知ることが第一歩】**
```
【情報がない状態】
「このままでいいのか...」(漠然とした不安)
 ↓
【情報を持った状態】
「今の会社に残る」or「転職する」(明確な判断)

情報を持つだけで、不安が減ります。
```

**【ポイント3: 今すぐ転職しなくてもOK】**
```
「今すぐ転職しなきゃいけない」
そんなことはありません。

大事なのは、
「情報を持った上で、自分で決める」こと。

✓ 情報収集だけでもOK
✓ 市場価値を聞くだけでもOK
✓ キャリア相談だけでもOK

転職するかどうかは、
情報を知ってから決めればいい。
```

**【図解】**
```
【間違ったステップ】
① 「転職しよう!」と決める
② エージェントに登録
③ 求人を探す

【正しいステップ】
① まず市場価値を知る ← ここから!
② キャリアの選択肢を知る
③ 「転職する」or「残る」を判断
④ 転職する場合、求人を探す

---

焦って決めなくていい。
まずは、情報を持つことから。
```

### **6. 手段の教育(今すぐ転職じゃない、つながりを持つ)**

#### **目的**
- 「エージェント = つながり」という新しい価値提案
- 行動のハードルを極限まで下げる
- 「押し売りされない」安心感を与える

#### **構成**

**【見出し】**
```
まずは「つながり」を持つだけでOK
エージェントは、キャリアの壁打ち相手として使える
```

**【手段1: つながりを持つメリット】**
```
エージェントに登録 = 転職
じゃないんです。

エージェントは、
「キャリアの壁打ち相手」として使えます。

✅ 今の市場価値を教えてもらう
✅ キャリアの選択肢を知る
✅ 「今動くべきか」を一緒に考える
✅ いい求人が出たら教えてもらう

つながりを持つだけで、
キャリアの不安が減ります。

転職するかどうかは、その時に考えればいい。
```

**【手段2: 4つの使い方】**
```
【使い方1: 市場価値診断】
「自分のスキルって、どのくらいの価値がある?」
→ プロの視点で、市場価値を診断

【使い方2: キャリア相談】
「このままのキャリアでいいのか?」
→ 技術トレンドを踏まえたアドバイス

【使い方3: 情報収集】
「今、どんな求人がある?」
→ 転職しなくても、情報だけ聞ける

【使い方4: タイミング判断】
「今動くべき? 待つべき?」
→ 正直にアドバイス(動かない方がいい場合も伝える)
```

**【手段3: 具体的なステップ】**
```
【ステップ1】
エージェントに登録(3分)

【ステップ2】
面談で、市場価値・キャリアの選択肢を聞く(1時間)

【ステップ3】
情報を持った上で、「今の会社に残る」or「転職する」を判断

【ステップ4】
転職する場合 → 求人紹介・面接対策をサポート
転職しない場合 → つながりを持っておく(いつでも相談OK)

---

転職するかどうかは、ステップ3で決めればいい。
まずは、情報を持つことから始めましょう。
```

### **7. 信用の教育(エンジニア専門だから技術が分かる)**

#### **目的**
- このサービスを選ぶ理由を明確にする
- 競合との差別化
- 「技術が分かる」ことの価値を伝える

#### **構成**

**【見出し】**
```
なぜ、エンジニア専門のエージェントなのか?
```

**【理由1: アドバイザーが技術に詳しい】**
```
「総合型の転職エージェント、使ったことあるけど...」
「アドバイザーが技術のこと全然分かってなくて、話が通じなかった」

そんな経験、ありませんか?

[サービス名]のアドバイザーは、
✅ エンジニア経験者
✅ または技術知識が豊富

だから、技術の話が通じます。

「Reactの経験があります」
「Docker使ってます」
「マイクロサービスで開発してます」

こういう話が、きちんと理解できる。

技術に詳しいアドバイザーだから、
本当にマッチする求人を紹介できます。
```

**【理由2: 開発環境・技術スタックまで把握】**
```
【他社のエージェント】
「この会社、給料いいですよ!」(表面的)

【[サービス名]】
「この会社、給料もいいし、
 技術スタックは○○、開発環境は○○、
 技術的負債は少なめ、チーム体制は○○人です」

ここまで詳しく教えられるのは、
エンジニア専門だから。

入社後のミスマッチを防げます。
```

**【理由3: エンジニア特化のサポート】**
```
[サービス名]では、
一般的な面接対策だけでなく、
エンジニア特化のサポートを提供します。

✅ ポートフォリオ添削
✅ GitHubレビュー
✅ 技術面接対策
✅ 技術ブログの書き方アドバイス

「技術的に何をアピールすればいいか」
これを一緒に考えます。
```

**【理由4: 押し売りしないスタンス】**
```
「転職エージェントって、押し売りされそう...」

安心してください。

[サービス名]は、
「今すぐ転職しなくてもOK」のスタンス。

無理に転職を勧めることはしません。

あなたのキャリアを一緒に考える、
壁打ち相手として使ってください。

「今は動かない方がいい」
そう判断したら、正直に伝えます。
```

**【差別化表】**
```
| 項目 | 他社エージェント | [サービス名] |
|------|--------------|----------|
| 技術理解 | 営業出身が多い | エンジニア経験者 |
| 求人紹介 | 表面的情報のみ | 技術スタックまで詳細 |
| サポート | 一般的な面接対策 | 技術面接・ポートフォリオ |
| スタンス | 早く転職させたい | 今すぐ転職じゃなくてOK |
```

### **9. 成功事例(複数の転職体験談)**

#### **目的**
- 信頼性を高める
- 具体的なイメージを持たせる
- 「自分も使ってみようかな」と思わせる

#### **構成**

**【見出し】**
```
実際に利用された方の声
```

**【体験談3: 27歳・年収アップ】**
```
「市場価値を知って、年収が200万円上がりました」

「このままでいいのか不安」で登録しました。

面談で市場価値を聞いたら、
「あなたのスキルなら、年収+200万円は狙えます」と。

正直、半信半疑でしたが、
実際に転職したら年収が200万円アップ。

「自分の市場価値、こんなに高かったんだ」
と驚きました。

(27歳・Web系エンジニア・年収400万→600万円)
```

**【体験談4: 31歳・環境改善】**
```
「レガシーな環境から、モダンな環境へ転職できました」

SESで受託開発ばかりで、スキルが伸びず...
「このままじゃマズい」と思って相談しました。

アドバイザーが技術に詳しくて、
「この会社なら、○○の技術が学べます」
と具体的に教えてくれた。

転職後、React、TypeScript、Dockerを使った開発ができて、
毎日が楽しいです。

(31歳・Web系エンジニア)
```

**【体験談5: 34歳・相談だけで不安が減った】**
```
「転職しませんでしたが、不安がなくなりました」

「34歳で転職、もう遅いかな...」と思って相談しました。

でも、アドバイザーに
「今の会社、待遇も環境も良いですよ。
 無理に転職しなくても大丈夫です」
と言ってもらえて。

情報を知った上で、今の会社に残ることにしました。

漠然とした不安がなくなって、
今の仕事に集中できています。

(34歳・インフラエンジニア)
```

### **11. オファー・CTA(行動の教育)**

#### **目的**
- 最後の背中押し
- 今すぐ行動させる
- 「このまま」vs「情報を持つ」の対比

#### **構成**

**【見出し】**
```
今日から、キャリアの第一歩を踏み出しませんか?
```

**【最後のメッセージ】**
```
「このままでいいのか...」

その不安の正体は、
「情報がない」からです。

情報があれば、決断できます。

だから、まずは情報を持ちましょう。

---

登録3分。
それだけで、キャリアの不安が減ります。

転職するかどうかは、その後決めればいい。

---

1年後、後悔しないために。

今日から、キャリアの第一歩を踏み出しませんか?
```

**【CTA】**
```
[登録3分、相談無料]
まずは市場価値を知る →

✓ 登録無料
✓ 今すぐ転職しなくてもOK
✓ 押し売りなし
```

**【最後の背中押し(対比)】**
```
今、あなたには2つの選択肢があります。

【選択肢A】
「このまま」何もせず、様子を見る
→ 1年後、「あの時動いておけば...」と後悔

【選択肢B】
まずは情報を持ってから、判断する
→ 1年後、「情報を持った上で決めた」と納得

どちらを選びますか?

---

選ぶのは、今のあなたです。
```

- 押し売りしないスタンス
